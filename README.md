# SkyGuardian

Административная система для получения новостей из Telegram-источников и публикации в выбранные каналы.

## Механизм новостей

1. В **Новости → Настройка** создаётся Telegram App с API ID и API Hash.
2. К App добавляется технический аккаунт и один раз авторизуется по телефону или QR-коду.
3. В **Новости → Каналы данных** создаётся связка «источник → техаккаунт → канал публикации».
4. `news:work` проверяет каждый активный источник по его интервалу от 3 секунд до 12 часов.
5. Новые сообщения определяются по Telegram `message_id`, передаются в очередь и публикуются отдельным worker.

Старые сообщения при первом запуске не публикуются. Для каждого источника сохраняются только служебные ID, время и результат обработки — текст и медиа сообщений в базе не хранятся.

Если техаккаунт удалить, связанные каналы данных не удаляются: они остаются в списке в состоянии **«Отключён»** и могут быть назначены другому аккаунту.

## Защита и устойчивость

- API ID, API Hash, телефон и Telegram-сессия зашифрованы ключом Laravel.
- Код Telegram и пароль 2FA не сохраняются.
- Один аккаунт защищён общей блокировкой от параллельных проверок, публикаций и авторизаций.
- Задание публикации записывается до обновления `last_message_id`.
- Дубли блокируются связкой источника, назначения и Telegram ID.
- `FloodWait` Telegram имеет приоритет над пользовательским интервалом.
- Неудачные публикации повторяются с задержками 10, 30, 60 и 300 секунд.
- Альбомы с одним `grouped_id` обрабатываются как одна публикация.

## Локальный запуск

Требуются PHP 8.2+, Composer, PostgreSQL или SQLite и Node.js.

```bash
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate
npm ci
npm run build
php artisan serve
```

В отдельных процессах запускаются:

```bash
php artisan news:work --sleep=1
php artisan queue:work database --queue=news,default --tries=5 --timeout=300
```

В production оба процесса управляются `systemd` и автоматически перезапускаются после сбоя или перезагрузки сервера.
